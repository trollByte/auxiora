# Deep Research Mode Design

**Date:** 2026-02-20
**Status:** Approved

## Goal

Add an autonomous deep research mode that decomposes complex questions into subtopics, executes parallel multi-source research with iterative refinement, and produces structured reports or conversational summaries with citations and confidence scores.

## Decisions

- **Approach:** Plan-Execute-Refine loop (LLM plans subtopics, parallel research, gap detection, 1-2 refinement rounds)
- **Package:** Extend `packages/research/` (no new package)
- **Output:** Mode-dependent — structured report for deep, conversational summary for quick/standard
- **Triggering:** Auto-detect research intent + explicit command/API
- **Progress:** Real-time WebSocket streaming of pipeline stages
- **Storage:** In-memory job map (1-hour expiry), no vault persistence

## Section 1: Core Architecture

### DeepResearchOrchestrator

New class in `packages/research/src/deep-research.ts`.

Pipeline stages:

```
Question → Plan → Execute → Evaluate → (Refine?) → Synthesize → Report
```

1. **Plan**: LLM decomposes the question into 3-8 subtopics, each with search queries and focus areas
2. **Execute**: For each subtopic, call `ResearchEngine.research()` in parallel (respecting `maxConcurrentSources`). Ingest findings into `DocumentStore` for RAG retrieval.
3. **Evaluate**: LLM reviews collected findings against the original question, identifies knowledge gaps (missing subtopics, conflicting sources, low-confidence areas)
4. **Refine** (0-2 rounds): Generate targeted follow-up queries for gaps, execute, re-evaluate
5. **Synthesize**: Build final report using RAG context from all findings + KnowledgeGraph entities
6. **Report**: Generate structured output (sections, citations, confidence) or conversational summary

### Configuration

```ts
interface DeepResearchConfig {
  maxSubtopics: number;        // default 6
  maxRefinementRounds: number; // default 2
  maxTotalSources: number;     // default 20
  tokenBudget: number;         // default 100_000
  timeoutMs: number;           // default 300_000 (5 min)
}
```

## Section 2: Auto-Detection & Triggering

### ResearchIntentDetector

New class in `packages/research/src/intent-detector.ts`. Lightweight pattern-based classifier (no LLM call).

**Signals that increase research score:**
- Question words + complexity markers ("compare", "analyze", "pros and cons", "differ from")
- Multi-faceted questions (multiple entities or domains)
- Fact-seeking patterns ("current state of", "latest developments")
- Explicit research keywords ("research", "investigate", "deep dive")

**Signals that decrease score:**
- Simple factual questions
- Code/task requests
- Personal/conversational
- Follow-up in same chat context

**Output:**

```ts
interface ResearchIntent {
  score: number;           // 0-1
  suggestedDepth: 'quick' | 'standard' | 'deep';
  reason: string;
  subtopicHints: string[];
}
```

**Threshold:** Score >= 0.6 triggers `research_suggestion` WebSocket event. User accepts or dismisses. Explicit `/research` command bypasses detection.

**Runtime integration:** Called in `handleMessage()` before the LLM call.

## Section 3: Progress Streaming & Report Generation

### Progress Events

```ts
type ResearchProgressEvent =
  | { type: 'research_started'; questionId: string; subtopicCount: number }
  | { type: 'research_planning'; subtopics: string[] }
  | { type: 'research_searching'; subtopic: string; index: number; total: number }
  | { type: 'research_source_found'; subtopic: string; sourceCount: number }
  | { type: 'research_evaluating'; round: number; gapCount: number }
  | { type: 'research_refining'; round: number; newQueries: number }
  | { type: 'research_synthesizing'; findingCount: number; sourceCount: number }
  | { type: 'research_complete'; questionId: string; duration: number }
  | { type: 'research_failed'; questionId: string; error: string };
```

Emitted via `onProgress` callback. Runtime wires to WebSocket.

### Report Generator

New class in `packages/research/src/report-generator.ts`.

**Structured report (deep mode):**

```ts
interface ResearchReport {
  id: string;
  question: string;
  executiveSummary: string;
  sections: ReportSection[];
  knowledgeGaps: string[];
  sources: CitedSource[];
  metadata: {
    depth: ResearchDepth;
    totalSources: number;
    refinementRounds: number;
    duration: number;
    tokenUsage: number;
    confidence: number;
  };
}

interface ReportSection {
  title: string;
  summary: string;
  findings: string[];
  sources: string[];
  confidence: number;
}
```

**Conversational summary (quick/standard mode):** Single markdown string with inline citations, generated by AI provider using RAG context.

## Section 4: REST API & Audit

### Endpoints

Router mounted at `/api/v1/research`:

```
POST   /research              — Start a deep research job (async)
  Body: { question: string, depth?: 'quick'|'standard'|'deep', focusAreas?: string[] }
  Returns: { jobId: string, status: 'planning' }

GET    /research/:jobId        — Get job status and results
GET    /research               — List recent research jobs (?limit=20&offset=0)
DELETE /research/:jobId         — Cancel a running research job
GET    /research/:jobId/sources — Get all sources with credibility details
```

### WebSocket Handler

```
{ type: 'start_research', question, depth? }
```

Starts research and streams progress events back on the same connection.

### Audit Events

| Event | When |
|-------|------|
| `research.started` | Job begins (question, depth) |
| `research.completed` | Job finishes (jobId, sourceCount, duration) |
| `research.failed` | Job errors out |
| `research.cancelled` | User cancels running job |

### Job Storage

In-memory `Map<string, ResearchJob>` with job state, progress, and final report. Jobs expire after 1 hour. No vault persistence.

## What This Does NOT Change

- The existing ResearchEngine — used as-is by the orchestrator
- The RAG DocumentStore or Chunker — used as-is for ingestion
- The KnowledgeGraph — used as-is for entity linking
- The existing `research` tool definition — still works for simple queries
- Browser tools — not used by deep research (ResearchEngine handles fetching)
